# AI-планер дня — HANDOFF (передача в новий чат)

> Стислий, але повний стан проєкту. Прочитай цей файл першим у новому чаті.
> Дата останнього оновлення: 22 липня 2026.

## Що це
Mobile-first застосунок, що **планує весь день** (робота, спорт, дозвілля, сімʼя, побут — не лише «робочі» задачі). Користувач диктує/пише все, що в голові → **Claude (Haiku 4.5)** перетворює це на структуровані задачі. Це фінальне завдання курсу **SKELAR**.

Власниця — **не технічна**. Пояснювати простими словами. Кредити/паролі/токени вводить сама. **Пуш робить власниця через GitHub Desktop** (я лише комічу локально).

## Стек і структура
- **Next.js 15.5** (App Router), **React 19**, **TypeScript**, чистий CSS. Стан **тільки на клієнті** (`localStorage`), бекенду немає. Єдиний серверний шматок: `app/api/parse/route.ts` (виклик Claude).
- Екрани: `app/capture|inbox|today|history|welcome/page.tsx` (+ `triage` існує, але не в навігації).
  - `app/page.tsx` — гейт першого запуску: онбординг не пройдено → `/welcome`, інакше → `/capture`.
- Логіка/стан: `lib/useTasks.ts` (задачі, бюджет-графік дня, історія, авто-план), `lib/useProfile.ts` (імʼя + прапорець онбордингу), `lib/types.ts` (схема Task), `lib/i18n.ts` + `lib/LanguageContext.tsx` (UA/EN).
- Дизайн: `components/Mascot.tsx` (маскот «Ладо»), `components/Icon.tsx`, `components/TabBar.tsx`.

## Мова інтерфейсу
**Тільки українська + англійська.** Чат із власницею — російською, але в UI російська НЕ потрапляє ніколи (усі рядки — через `lib/i18n.ts`, є лише `uk`/`en`). Напр. «Всі», не «Все».

## Схема задачі (контракт AI↔UI)
`id, title, priority(low|medium|high), category(work|sport|leisure|family|chores|other), estimateMin(5/15/30/60/120|null), deadline(YYYY-MM-DD|null), status(inbox|today|done), scheduledFor(YYYY-MM-DD|null), suggested, createdAt`
- **AI** ставить: title, priority, category, estimateMin, deadline (лише якщо названо; **не вигадує**), isToday, suggested (топ-3).
- **Код** ставить status/scheduledFor.

## Ключова логіка (як задачі потрапляють у «Сьогодні»)
1. **При розборі** → у Today одразу, якщо: `isToday` (явне «сьогодні/терміново») АБО дедлайн ≤ сьогодні АБО `suggested` (рекомендована; ми їх авто-додаємо). Решта → беклог («Усі задачі»).
2. **Авто-наповнення з беклогу** — раз на новий день (при завантаженні, `LAST_PLANNED_KEY`): спершу задачі з дедлайном, що наближається (≤2 дні), потім за пріоритетом, поки не заповниться вікно дня. Задача без оцінки = 30 хв.
3. **Вручну** — «+ Сьогодні».
- **Дедуп:** розбір не додає мовчки дубль; якщо активна задача з такою ж назвою вже є (або повтор у дампі) → діалог «Такі задачі вже є — додати ще раз?».
- **Захист дат:** якщо в тексті дампу немає жодної згадки дати (`mentionsDate` у `route.ts`), усі дедлайни від AI відкидаються (страховка поверх промпта).

## Графік дня (замість «часу сну»)
`useDayBudget` зберігає **вікно планування** `planStartMin`/`planEndMin` (дефолт 09:00–22:00, ключі `ai-planner:plan-start-min`/`-end-min`). Доступний час = від `max(зараз, старт)` до кінця. Задається в онбордингу; кінець дня редагується степпером на «Сьогодні».

## Що вже зроблено (фічі)
- Capture (текст + **голос** укр.) → AI-розбір; велика центральна кнопка мікрофона; персональне привітання за часом доби + імʼям (кличний відмінок, напр. «Ксюшо»).
- **Онбординг** `/welcome`: імʼя + графік дня; показується раз.
- **Усі задачі** (беклог): фільтри-чипи за категорією (одиночний вибір, лише присутні, з лічильниками); редагований **дедлайн** (нативний date-input) і **пріоритет** (тап циклює).
- **Сьогодні**: прогрес, навантаження дня (проти вікна годин), розбивка по категоріях, сортування за пріоритетом, редагований пріоритет/час, святковий стан, «Завершити день» → перенос на завтра / у беклог, секція 🌙 Завтра, історія (`/history`).
- **UA/EN** перемикач; пріоритети (укр.): 🔥 Палає / Важливо / При нагоді.
- Дизайн-редизайн («живий мальований» + маскот) — влитий у main, на проді.

## Робочі принципи
- **Швидкість передусім, мінімальний флоу:** дрібні правки (текст/стиль/невелика логіка) — без спеків/планів, одразу код → показ → власниця пушить. Спек/план — лише на велике/ризиковане (напр. авторизація).
- Маленькі часті коміти: один фокусний коміт = один пуш = один деплой.
- Схема даних — головний контракт; продумувати словами перед кодом.
- Порожні екрани не мають виглядати як баг (скрізь онбординг-стани).
- Mobile-first, великі тач-зони, тестувати на телефоні (у Chrome для голосу).
- `ANTHROPIC_API_KEY` — у Vercel Env Vars (і локально `.env.local`); власниця вводить сама.

## Відкриті питання / ідеї на потім
- Дедуп ловить лише точні збіги назв (регістр/пробіли), не синоніми.
- Кличний відмінок імені — приблизний (типові укр. закінчення; латиниця/рідкісні — без змін).
- Можливий гуманізований показ дедлайну («завтра / за 2 дні» замість ISO).
- Авторизація + бекенд (див. роадмап).
