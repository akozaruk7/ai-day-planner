# Спек: фільтри у «Вхідних» (Inbox category filters)

> Дата: 22 липня 2026. Статус: затверджено власницею, готово до плану реалізації.
> Мова спека — робоча (рос./укр.). **Рядки інтерфейсу — тільки UA + EN** (див. правило нижче).

## Проблема / мета
У «Вхідних» ([app/inbox/page.tsx](../../../app/inbox/page.tsx)) зараз плоский список без фільтрів. Коли задач багато, важко подивитись лише один рід діяльності (робота / спорт / побут …). Мета — дати швидкий фільтр за категорією над списком.

## Не-цілі (YAGNI)
- Не фільтруємо за пріоритетом (лише категорія).
- Не додаємо фільтри на екран «Сьогодні» (тільки «Вхідні»).
- Не змінюємо модель даних / `useTasks` — фільтр суто візуальний.
- Не запам'ятовуємо вибраний фільтр між заходами.
- Множинний вибір не робимо.

## Дизайн

### Вигляд
- Горизонтальний ряд чипів **над списком** задач.
- Чипи: **«Всі»** + по одному на кожну категорію, що **реально присутня** у поточному списку Inbox, у сталому порядку `CATEGORIES` (work → sport → leisure → family → chores → other).
- На кожному чипі — назва + **лічильник** (напр. `Робота 3`). На «Всі» — загальна кількість.
- Ряд горизонтально скролиться, якщо чипи не влазять (mobile-first).
- Активний чип підсвічений; стиль — у тому ж «мальованому» вигляді, що й наявні теги категорій (`.cat--*`).

### Поведінка (одиночний вибір)
- За замовчуванням активна **«Всі»** — показані всі задачі Inbox.
- Тап по чипу категорії → список показує лише цю категорію, чип стає активним.
- Тап по «Всі» → скидання.
- Стан фільтра — локальний стан екрана (`useState`), не персиститься; при виході з вкладки скидається на «Всі».

### Крайні випадки
- Чипи показують лише присутні категорії → вибраний фільтр завжди дає ≥1 задачу; окремий «порожній під фільтр» стан не потрібен.
- Якщо остання задача вибраної категорії пішла в «Сьогодні» / стала виконаною і категорія зникла зі списку → авто-скидання на «Всі».
- Коли Inbox порожній зовсім → ряд фільтрів приховано, показується наявний онбординг-порожній стан.

### Доступність
- Чипи — це `<button>` з `aria-pressed` для активного; зрозумілі тап-зони (mobile-first).

## Що торкаємо (3 файли)
- [app/inbox/page.tsx](../../../app/inbox/page.tsx) — ряд чипів, локальний стан вибору, фільтрація відображеного списку, авто-скидання.
- [app/globals.css](../../../app/globals.css) — стилі `.filter-chip` / активний стан у наявному стилі.
- [lib/i18n.ts](../../../lib/i18n.ts) — новий рядок «Всі» / «All» (назви категорій уже є в `t.cat`).

## Рядки інтерфейсу (тільки UA/EN — правило проекту)
- `all`: UA **«Всі»**, EN **«All»**. (НЕ російське «Все».)
- Лічильники — числа, без нових слів.

## Перевірка (verification)
- `npm run build` = exit 0 (правило: білд перед пушем).
- Ручна перевірка в браузері (mobile): тап по чипу фільтрує; «Всі» скидає; лічильники правильні; авто-скидання коли категорія зникла; порожній Inbox ховає ряд.

## Поза обсягом / майбутнє
- Фільтр за пріоритетом; фільтри на «Сьогодні»; запам'ятовування вибору.
